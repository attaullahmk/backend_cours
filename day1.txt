üöÄ Types of Functions in JavaScript
JavaScript gives multiple ways to create functions for different situations.


1Ô∏è‚É£ Function Declaration (Standard Function)

üìå Syntax

‚úî Why Use It?

‚úÖ Question 3: What is the difference between var, let, and const in JavaScript?
üîπ Definition:
var, let, and const are used to declare variables in JavaScript. But they differ in scope, hoisting, and reassignment rules.

| Feature                 | `var`           | `let`           | `const`         |
| ----------------------- | --------------- | --------------- | --------------- |
| Scope                   | Function        | Block           | Block           |
| Hoisting                | Yes (undefined) | Yes (TDZ error) | Yes (TDZ error) |
| Redeclaration           | ‚úÖ Yes           | ‚ùå No            | ‚ùå No            |
| Reassignment            | ‚úÖ Yes           | ‚úÖ Yes           | ‚ùå No            |
| Initialization Required | ‚ùå No            | ‚ùå No            | ‚úÖ Yes           |

"Pehle JavaScript mein var use hota tha, lekin us mein scope aur hoisting ke issues thay. ES6 mein let aur const aye jo block-level scope dete hain. let tab use karte hain jab value change ho sakti ho. const tab use karte hain jab value fix ho aur sirf ek dafa assign karni ho."







‚úÖ Question 4: What is hoisting in JavaScript?
üîπ Definition:
Hoisting is JavaScript's default behavior of moving declarations to the top of their scope (function or global) before the code is executed. 


In simple words: JavaScript ‚Äúremembers‚Äù your variable or function declarations at the top of the scope ‚Äî but not their assignments.

üî∏ Hoisting with let and const
console.log(b); // ‚ùå ReferenceError
let b = 20;

Why? Because of the Temporal Dead Zone (TDZ) ‚Äî time between hoisting and actual declaration where variable exists but can‚Äôt be used.

üî∏ Hoisting with Functions
Function declarations are hoisted fully ‚Äî both the name and the body.
greet(); // Output: Hello
function greet() {
  console.log("Hello");
}

But function expressions are only partially hoisted:
sayHi(); // ‚ùå TypeError: sayHi is not a function

var sayHi = function() {
  console.log("Hi");
};

Here only var sayHi is hoisted (undefined), not the function part.

| Type                   | Hoisted?        | Initialized?          |
| ---------------------- | --------------- | --------------------- |
| `var`                  | ‚úÖ Yes           | ‚úÖ With `undefined`    |
| `let` / `const`        | ‚úÖ Yes           | ‚ùå No (TDZ applies)    |
| Function (declaration) | ‚úÖ Yes           | ‚úÖ Yes (fully hoisted) |
| Function expression    | ‚úÖ Variable only | ‚ùå Not initialized     |


"Hoisting ka matlab hai ke JavaScript pehle variables aur functions ko memory mein upar le jaata hai. var hoist hota hai lekin undefined assign hota hai. let aur const bhi hoist hote hain, lekin unhein access karna declaration se pehle error deta hai ‚Äî isay Temporal Dead Zone kehte hain."
